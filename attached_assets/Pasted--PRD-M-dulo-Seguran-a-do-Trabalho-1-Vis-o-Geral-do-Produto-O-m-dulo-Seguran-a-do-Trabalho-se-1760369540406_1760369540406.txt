🧩 PRD — Módulo “Segurança do Trabalho”
1. Visão Geral do Produto

O módulo Segurança do Trabalho será uma nova aba da plataforma EcoBrasil Gestão Corporativa, integrada ao sistema principal e ao Dashboard Geral.
Seu objetivo é permitir a gestão documental e preventiva de segurança ocupacional, com acompanhamento de conformidade por colaborador, setor e empreendimento.

2. Objetivos Principais

Centralizar documentos assinados de segurança do trabalho por colaborador.

Acompanhar indicadores de conformidade por empreendimento e geral.

Permitir upload, controle de validade e rastreabilidade de documentos obrigatórios.

Integrar os dados ao Dashboard Geral e Painel de Empreendimentos, exibindo:

Percentual de colaboradores conformes;

Documentos válidos x vencidos;

Alertas automáticos de vencimento;

Indicadores consolidados (por empreendimento e total).

3. Escopo Funcional
3.1. Estrutura da Aba “Segurança do Trabalho”

A nova aba conterá três submódulos principais:

🔹 1. Documentos Assinados

Upload de documentos PDF (ASO, treinamento, EPI, laudos etc.);

Vinculação ao colaborador e empreendimento;

Campos obrigatórios:

Tipo de documento (dropdown: ASO, Treinamento NR, EPI, LTCAT, PCMSO, etc.);

Data de emissão;

Data de validade (opcional);

Responsável técnico / assinatura digital;

Observações;

Status automático: Válido, Vencido ou Pendente;

Preview do PDF em modal;

Download direto do arquivo.

🔹 2. Indicadores de Conformidade

Exibição em cards e gráficos (por empreendimento e total):

Total de documentos cadastrados;

% de documentos válidos;

% de colaboradores conformes;

Nº de documentos vencidos;

Nº de colaboradores com pendências;

Filtro por:

Empreendimento;

Tipo de documento;

Período (últimos 6/12 meses).

🔹 3. Alertas e Acompanhamento

Geração automática de alertas:

Documentos a vencer (30 dias antes);

Documentos vencidos;

Envio de e-mail ou notificação interna;

Lista de pendências com destaque visual (badge vermelha).

4. Integração com o Dashboard Geral

No Dashboard Geral, incluir um bloco adicional “Segurança do Trabalho”, contendo:

Indicador	Tipo	Fonte de dados
% de conformidade geral	Card + gauge	Todos os colaboradores ativos
Nº de documentos válidos/vencidos	Card + gráfico de barras	Tabela seg_documentos_colaboradores
Top 5 empreendimentos mais conformes	Tabela resumida	Agregado por empreendimento
Alerta de vencimentos próximos	Lista (últimos 30 dias)	Query de validade

Além disso, no Painel por Empreendimento, incluir uma aba lateral:
📁 Segurança do Trabalho → Documentos / Indicadores / Alertas

5. Estrutura de Dados (Banco)
Tabela: seg_documentos_colaboradores
Campo	Tipo	Descrição
id	int (PK)	Identificador
colaboradorId	int (FK)	Referência ao colaborador
empreendimentoId	int (FK)	Referência ao empreendimento
tipoDocumento	varchar(100)	Ex: “ASO”, “Treinamento NR-33”, “Entrega EPI”
descricao	text	Detalhes adicionais
arquivoUrl	varchar(255)	Link do arquivo PDF
dataEmissao	date	Data de emissão
dataValidade	date	Data de validade
assinaturaResponsavel	varchar(255)	Nome ou ID do responsável técnico
status	enum(“válido”, “vencido”, “pendente”)	Atualizado automaticamente
criadoEm	timestamp	Registro de criação
Relações

colaboradorId → tabela colaboradores

empreendimentoId → tabela empreendimentos

6. Requisitos Técnicos
Camada	Recurso	Descrição
Frontend	React + shadcn/ui	Página “Segurança do Trabalho” e integração com Dashboard
Backend	Express + Drizzle ORM	Rotas REST e controle de status documental
Banco	PostgreSQL	Armazenamento das tabelas seg_documentos_colaboradores
Upload	Multer / Cloud Storage (AWS ou supabase)	Armazenamento dos PDFs assinados
Visualização	react-pdf	Exibição dos documentos dentro da aplicação
Gráficos	recharts	Indicadores de conformidade e vencimento
7. Rotas Backend
Método	Endpoint	Função
POST	/api/seguranca/documentos	Cria novo documento vinculado a colaborador
GET	/api/seguranca/documentos/:id	Consulta documentos de um colaborador
GET	/api/seguranca/indicadores	Retorna indicadores globais e por empreendimento
DELETE	/api/seguranca/documentos/:id	Remove um documento
PATCH	/api/seguranca/documentos/:id	Atualiza status ou validade
8. Indicadores e KPIs Monitorados
Indicador	Cálculo	Meta
% de Conformidade Geral	(Documentos válidos ÷ Total de documentos) × 100	≥ 95%
% de Colaboradores Conformes	(Colaboradores com todos docs válidos ÷ Total) × 100	≥ 90%
Documentos a Vencer (30 dias)	Query por dataValidade < hoje+30	-
Documentos Vencidos	Query por dataValidade < hoje	-
9. Regras de Negócio

Documentos sem data de validade são considerados “válidos indefinidamente”.

Status “vencido” é atualizado automaticamente quando dataValidade < hoje.

O upload só é aceito nos formatos .pdf, .png ou .jpg.

O colaborador deve existir previamente no sistema (FK obrigatória).

A exclusão de documento exige permissão de usuário administrador.

Os documentos e status devem alimentar o Painel de Indicadores ESG futuramente.
