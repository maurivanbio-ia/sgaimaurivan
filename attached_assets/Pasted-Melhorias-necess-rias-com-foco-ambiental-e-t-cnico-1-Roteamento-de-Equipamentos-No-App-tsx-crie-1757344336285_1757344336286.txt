Melhorias necessárias (com foco ambiental e técnico)
1. Roteamento de Equipamentos

No App.tsx, crie rotas que delegam para o módulo:

/equipamentos → lista de equipamentos.

/equipamentos/:id → detalhe de um equipamento.

/equipamentos/painel → painel ambiental (KPIs, gráficos ESG).

Use lazy(() => import("@/pages/equipamentos")) para carregamento dinâmico.

2. Lista de equipamentos (equipamentos.tsx)

Filtros ambientais: adicionar selects para bioma, categoria, status e empreendimento.

Alertas:

Mostrar quantos equipamentos têm manutenção ou calibração vencida.

Mostrar quantos vencem nos próximos 30 dias.

Exportação CSV:

Permitir baixar inventário, com opção de ofuscar dados sensíveis (ex.: patrimônio de equipamento usado em fauna ameaçada).

Badges ambientais:

Adicionar etiqueta “dados sensíveis” em equipamentos associados a espécies ou campanhas restritas.

Colorir proximidade de manutenção/calibração:

Verde (ok), Amarelo (≤30 dias), Vermelho (vencido).

Link para painel ESG: botão “Abrir Painel ESG” → /equipamentos/painel.

3. Painel ESG (equipment-dashboard.tsx)

Criar cards de indicadores ambientais:

Número total de equipamentos ativos.

Quantos vencidos / próximos de vencer.

MTBF (tempo médio entre falhas).

MTTR (tempo médio de reparo).

Usar Recharts para gráfico de distribuição por bioma e categoria.

Adicionar seção “Impacto Ambiental” com resumo textual:

Ex.: “70% dos equipamentos de fauna estão calibrados e dentro do prazo”.

4. Detalhe de equipamento (equipamento-detail.tsx)

Mostrar campos ambientais extras:

Bioma, categoria, empreendimento associado.

Datas de manutenção/calibração com status de proximidade.

Adicionar abas:

Geral: dados básicos.

Histórico ambiental: logs de manutenção, calibração, uso em campanhas de campo.

ESG: indicador se o equipamento atende normas (ISO, IBAMA etc.).

Ações: Editar, Gerar QR Code, Voltar para lista.

5. Header (Header.tsx)

Melhorar isActive: tratar querystring e hash (ignorar).

Usar Button asChild para envolver Link e evitar <a><button></button></a>.

Adicionar aria-current="page" nos links ativos (acessibilidade).

Trocar logo para HTTPS ou asset local para evitar bloqueio em produção.

No menu mobile, adicionar os mesmos itens + botão de logout.
