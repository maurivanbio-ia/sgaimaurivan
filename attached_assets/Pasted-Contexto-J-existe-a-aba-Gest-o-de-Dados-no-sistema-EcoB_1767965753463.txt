Contexto: Já existe a aba “Gestão de Dados” no sistema EcoBrasil. NÃO criar uma aba nova. NÃO refazer a UI inteira. Implementar tudo como extensão incremental nessa aba existente, reaproveitando componentes, rotas, estilo, autenticação e banco atuais.

Objetivo: Na aba existente “Gestão de Dados”, implementar:
1) Inicialização e manutenção da estrutura macro de pastas institucional.
2) Criação automática da estrutura padrão por projeto quando um projeto for cadastrado (ou ao primeiro upload).
3) Formulário de upload com metadados obrigatórios para gerar automaticamente o código do arquivo (nome padronizado).
4) Roteamento automático do arquivo para a pasta correta dentro do projeto, baseado em DOC, STATUS e extensão.
5) Seção fixa “Dicionário de Siglas” (curta, pesquisável) dentro da própria aba.
6) Rodapé fixo com texto normativo (ISO + FAIR + LGPD).
7) Controle de versões e trilha de auditoria (audit trail) para cada documento, mantendo histórico.

Diretrizes obrigatórias:
- Prefixo fixo do código: ECOBRASIL
- Nome do arquivo gerado automaticamente:
  ECOBRASIL-[CLIENTE]-[UF]-[PROJ]-[SUBPROJ]-[DISC]-[ENTREGA]-[DOC]-[AREA]-[PERIODO]-[DATA]-[RESP]-[Vx.y]-[STATUS].[ext]
- Campos opcionais vazios devem ser omitidos sem deixar separadores duplicados.
- Normalização: remover acentos, remover caracteres especiais, espaço -> underscore, tudo em maiúsculas.
- DATA no padrão AAAAMMDD. default no dia do upload.
- Versão inicia em V0.1, e incrementa se já existir o mesmo código.
- Armazenar hash (SHA-256) e timestamps para integridade e rastreabilidade.
- Não quebrar o que já funciona. criar migrações compatíveis.

Tarefa 4: Roteamento automático (pasta destino)
- Definir função que retorna destino com base em DOC, STATUS e extensão:
