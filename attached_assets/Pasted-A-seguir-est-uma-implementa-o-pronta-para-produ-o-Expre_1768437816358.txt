A seguir está uma implementação pronta para produção (Express + TypeScript) criando exatamente:

. GET /api/licencas/:id
. PUT /api/licencas/:id
. POST /api/upload/pdf em dois modos no mesmo endpoint
. Modo 1. application/json para gerar parâmetros { method, url, filePath } (compatível com seu ObjectUploader)
. Modo 2. multipart/form-data para receber o arquivo e salvar no filePath informado

Isso resolve o seu problema sem “adivinhar storage externo”. Se no futuro você migrar para S3/R2, eu adapto para presigned URL real, mas o contrato com o front permanece igual.

1.3 src/routes/uploadPdf.ts

Este endpoint é o que seu ObjectUploader quer. Primeiro o front chama para obter parâmetros. Depois ele faz o upload para a url retornada.

import { Router } from "express";
import multer from "multer";
import path from "path";
import fs from "fs/promises";
import { z } from "zod";

const router = Router();

/* Pasta pública onde o arquivo final será servido.
   Garanta também um static em /uploads no app (abaixo). */
const UPLOAD_ROOT = path.resolve(process.cwd(), "public", "uploads", "licencas");

const presignSchema = z.object({
  filename: z.string().min(1).optional(),
  contentType: z.string().optional(),
});

/* Armazena em memória para permitir salvar exatamente no filePath solicitado */
const upload = multer({
  storage: multer.memoryStorage(),
  limits: {
    fileSize: 25 * 1024 * 1024, // 25 MB
  },
  fileFilter: (_req, file, cb) => {
    const ok = file.mimetype === "application/pdf" || file.originalname.toLowerCase().endsWith(".pdf");
    if (!ok) return cb(new Error("Apenas PDF é permitido"));
    return cb(null, true);
  },
});

function safeBasename(name: string) {
  const base = path.basename(name).replace(/[^\w.\-() ]+/g, "_");
  return base.toLowerCase().endsWith(".pdf") ? base : `${base}.pdf`;
}

async function ensureDir(dir: string) {
  await fs.mkdir(dir, { recursive: true });
}

/* ============================================================
   POST /api/upload/pdf
   Modo 1. application/json -> retorna { method, url, filePath }
   Modo 2. multipart/form-data -> recebe "file" e salva no filePath
   ============================================================ */
router.post("/pdf", async (req, res, next) => {
  const ct = String(req.headers["content-type"] ?? "");
  const isMultipart = ct.includes("multipart/form-data");

  if (isMultipart) return next();

  try {
    const parsed = presignSchema.safeParse(req.body ?? {});
    if (!parsed.success) {
      return res.status(400).json({ message: "Body inválido", issues: parsed.error.issues });
    }

    await ensureDir(UPLOAD_ROOT);

    const filename = safeBasename(parsed.data.filename ?? `licenca_${Date.now()}.pdf`);
    const filePath = `/uploads/licencas/${filename}`;

    /* O ObjectUploader vai fazer upload para esta mesma rota, passando o filePath como query. */
    const url = `/api/upload/pdf?filePath=${encodeURIComponent(filePath)}`;

    return res.json({
      method: "POST",
      url,
      filePath,
    });
  } catch (e: any) {
    return res.status(500).json({ message: e?.message ?? "Erro ao gerar parâmetros de upload" });
  }
});

/* Upload efetivo.
   Espera multipart com campo: file
   E query: filePath, retornado no passo anterior */
router.post("/pdf", upload.single("file"), async (req, res) => {
  try {
    const q = String(req.query.filePath ?? "").trim();
    if (!q.startsWith("/uploads/licencas/")) {
      return res.status(400).json({ message: "filePath inválido" });
    }

    if (!req.file) {
      return res.status(400).json({ message: "Arquivo não enviado. Campo esperado: file" });
    }

    const abs = path.join(process.cwd(), "public", q);
    await ensureDir(path.dirname(abs));
    await fs.writeFile(abs, req.file.buffer);

    return res.json({
      filePath: q,
      size: req.file.size,
      contentType: req.file.mimetype,
    });
  } catch (e: any) {
    return res.status(500).json({ message: e?.message ?? "Erro ao salvar PDF" });
  }
});

export default router;

2) Ajuste mínimo no seu componente de edição

Agora sim seu front pode manter:

. GET /api/licencas/:id
. PUT /api/licencas/:id
. POST /api/upload/pdf

No seu useQuery, você precisa de queryFn (no seu snippet estava faltando):

const { data: license, isLoading } = useQuery<LicencaAmbiental>({
  queryKey: ["/api/licencas", id],
  enabled: !!id,
  queryFn: async () => {
    const res = await apiRequest("GET", `/api/licencas/${id}`);
    return res.json();
  },
});


E no update:

const response = await apiRequest("PUT", `/api/licencas/${id}`, data);
return response.json();


E no uploader, você mantém o contrato, só que agora aponta para /api/upload/pdf que existe:

onGetUploadParameters={async () => {
  const response = await apiRequest("POST", "/api/upload/pdf", {
    filename: `licenca_${id}_${Date.now()}.pdf`,
    contentType: "application/pdf",
  });
  const data = await response.json();
  return { method: data.method, url: data.url, filePath: data.filePath };
}}

3) Observações críticas para “funcionar de verdade”

. Se o seu apiRequest já faz fetch e retorna Response, o .json() está correto. Se o seu apiRequest já retorna JSON direto, você deve remover .json().
. O ObjectUploader precisa enviar o arquivo no campo file. Se ele usa outro nome, ajuste upload.single("file") para o nome correto.
. O link href={license.arquivoPdf} vai funcionar porque agora você está servindo app.use("/uploads", express.static(...)).
